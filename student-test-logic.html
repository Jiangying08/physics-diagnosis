⚙️ 计分逻辑 Step 1
1
收集原始得分
遍历 12 道题，将每道题所选选项的 score 累加到对应维度的原始分 v 中，同时该维度满分 cnt 加 3。每道题仅贡献 1 个选项的分数。
2
换算为百分制
每个维度得分 = Math.round( v / cnt × 100 )，取值范围 0~100。其中 v 为原始得分总和，cnt 为该维度满分（题数 × 3）。
3
判定类型
将 5 个维度得分传入 detType(scores) 函数，按优先级顺序依次匹配，返回学员类型 A~E。
⚙️ 判型算法 Step 2

下面是完整的判型逻辑代码，按优先级从上到下依次判断：

// 输入：5 个维度得分 { concept, model, math, reading, flexible }
// 输出：类型 A / B / C / D / E

function detType(scores) {
const { concept, model, math, reading, flexible } = scores;
const avg = (concept + model + math + reading + flexible) / 5;


// 🥇 第 1 优先级：全面薄弱 → E 缓慢建构者
if (avg < 45) return 'E';


// 🥇 第 2 优先级：灵活但概念/建模偏低 → C 快速扫描者
if (flexible >= 60 && concept <= flexible - 15 && model <= flexible - 15)
return 'C';


// 🥇 第 3 优先级：概念+建模突出，审题偏低 → A 直觉建构者
if (concept >= 55 && model >= 55 && concept + model >= 120 && reading <= concept - 10)
return 'A';


// 🥇 第 4 优先级：数学+审题突出，灵活偏低 → B 规范执行者
if (math >= 55 && reading >= 55 && math + reading >= 120 && flexible <= math - 10)
return 'B';


// 🥇 第 5 优先级：数学偏高，概念偏低 → D 记忆驱动者
if (math >= 50 && concept <= math - 15)
return 'D';


// 🥈 备选：概念+建模突出（不限审题）→ A
if (concept >= 50 && model >= 50 && concept + model >= math + reading + flexible)
return 'A';


// 🥈 备选：数学+审题突出 → B
if (math >= 50 && reading >= 50)
return 'B';


// 🥈 备选：灵活突出 → C
if (flexible >= 55)
return 'C';


// 🥈 备选：数学最高 → D
if (math >= Math.max(concept, model, reading, flexible))
return 'D';


// 🥉 兜底：平均分低于 55 → E，否则 → A
if (avg < 55) return 'E';
return 'A';
}
📊 判型决策流程图
🥇 主规则（按顺序匹配，命中即返回）
① avg < 45 → E 缓慢建构者
② flexible ≥ 60 且概念/模型都 ≤ flexible−15 → C 快速扫描者
③ concept ≥ 55、model ≥ 55、concept+model ≥ 120、reading ≤ concept−10 → A 直觉建构者
④ math ≥ 55、reading ≥ 55、math+reading ≥ 120、flexible ≤ math−10 → B 规范执行者
⑤ math ≥ 50 且 concept ≤ math−15 → D 记忆驱动者

🥈 备选规则（主规则未命中时）
⑥ concept+model 之和最大 → A
⑦ math+reading 均 ≥ 50 → B
⑧ flexible ≥ 55 → C
⑨ math 是五项最高 → D

🥉 兜底
⑩ avg < 55 → E；否则 → A
